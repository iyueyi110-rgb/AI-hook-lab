# AI Hook Lab 一页产品策略

> 版本：2026-07-21。本文只陈述仓库可复核事实与待验证假设；当前没有可验证的用户访谈成果，离线评测和模拟数据不作为用户证据。

## 北极星指标

**真实用户任务采用率** = 有已确认采用结果的 `real_user` 完成任务数 ÷ 可关联任务 ID 的 `real_user` 完成任务数。

- “已确认采用”指用户在任务结束时明确选择直接使用或修改后使用；只生成、收藏、模型推荐或模型评分不进入分子。
- 分母仅包含来源为 `real_user`、状态为完成且可关联 `taskId` 的任务；无法关联、测试探针、离线评测和模拟任务不进入分母。
- 对外报告必须同时给出时间范围、Prompt 版本、分子、分母和样本量，样本不足时只展示计数，不展示百分比。
- 该指标衡量“生成结果是否进入创作者下一步工作”，不等于实际发布率、点击率或传播效果。

解释护栏：

| 护栏 | 用途 |
| --- | --- |
| 任务覆盖率 | 说明可关联 `taskId` 的完成任务占比，防止分母缺失造成虚高 |
| 反馈响应率 | 说明完成任务中有多少提供了明确采用结果，暴露选择性反馈 |
| 生成完成率 | 监控产品是否稳定完成生成，避免只优化内容而忽略可用性 |
| Bad Case 分布 | 按主题锚定、表达泛化、平台语气、标题党、事实可信度等标签解释失败原因 |

“实际发布采用率”作为更下游的长期指标，只有获得发布追踪授权并通过回访确认的任务进入其分母；不能用任务内选择替代发布采用。

## 差异定位

AI Hook Lab 不是通用文案平台的缩小版，而是中文多平台 Hook 专项工作台：用 10 候选比较降低单次生成的偶然性，用人工采用反馈连接真实创作任务，并用来源隔离和离线评测避免把模型自评分包装成用户效果。

当前最重要的产品问题不是继续增加生成模块，而是验证创作者是否愿意从候选中选择、修改并采用结果。

## Now / Next / Later

| 阶段 | 目标 | 完成标准 |
| --- | --- | --- |
| Now | 完成安全交付；招募 5–8 名近一个月持续创作的目标用户；用真实主题完成受控验证 | 建立不少于 30 个 `real_user` 完成任务的基线；分开报告任务覆盖率、反馈响应率、生成完成率、采用计数和 Bad Case 分布 |
| Next | 根据真实选择、改写和放弃原因推进跨设备创作资产与 Prompt 实验运营 | 每项改动绑定假设、版本、目标指标和停止条件；只有同口径证据进入版本比较 |
| Later | 评估发布效果回传与团队协作 | 先解决发布追踪授权、隐私和来源归因，再决定是否建设效果回传、品牌资产与多人工作流 |

## 当前证据边界与研究计划

- 当前仓库能证明 20 个主题 × 3 个平台的 60 个固定离线评测案例及其评测方法，不能证明真实用户采用或传播效果。
- 当前没有可验证的用户访谈、5–8 人测试或不少于 30 个真实完成任务；这些都是 Now 阶段待执行工作，不能写成已完成成果。
- 参与者、任务、记录字段、同意流程和判定方式见[真实创作者验证计划](real-user-validation-plan.md)；公开数字必须先通过[证据清单](../evidence/README.md)核验。
