# 真实创作者验证计划

## 目标

验证“候选看起来不错”能否转化为“创作者愿意为当前主题选择、修改并实际发布”，而不是继续增加离线模型分数。

## 样本与流程

- 招募 5–8 名近一个月持续创作、能说明主平台和内容类型的创作者。
- 每人使用自己的真实主题完成至少 3 个任务，目标累计不少于 30 个完成任务。
- 每个任务经历结构化输入、候选比较、最终选择、必要改写和发布意向确认。
- 记录直接使用、轻改、重写、放弃及原因；发布追踪需单独获得同意。

## 成功判断

- 先建立任务完成率、任务级选择率、反馈覆盖率和改写分布基线，不预设夸大的提升目标。
- 将“实际发布采用率”作为长期北极星候选，但只有获得发布追踪授权的样本进入分母。
- Bad Case 优先关注主题锚定不足、表达泛化、平台语气错配、标题党和事实可信度。
- 真实测试、离线评测和模拟数据继续按来源隔离，不跨来源合并百分比。

## Now / Next / Later

- Now：完成证据审计、公开 Demo 安全加固和 5–8 人验证。
- Next：根据真实选择与改写证据优化 Prompt/交互，再评估跨设备创作资产。
- Later：在获得授权后探索发布效果回传、团队协作和多模型成本/质量路由。
