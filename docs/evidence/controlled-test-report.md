# 创作者受控测试证据状态

## 当前结论

截至 2026-07-21，公开仓库仍未收到经产品负责人确认可公开的 `docs/evidence/data/controlled-test.csv`。因此，本报告不确认简历中曾出现的样本量和百分比，相关 manifest 状态继续保持 `not_verified`。

计算脚本与校验规则已经就绪：

```bash
npm run evidence:controlled
npm run evidence:verify
```

前一条命令在数据缺失时会明确失败，不会生成占位记录；后一条命令允许缺失数据的主张继续以 `not_verified` 公开，但禁止把对应数字写入产品与 portfolio 叙事。

## 公开数据契约

固定表头、字段约束和隐私边界见 [`data/README.md`](data/README.md)。产品负责人必须在提交前确认脱敏数据可以公开；Codex 只校验和计算，不反推或补齐事件。

收藏率以有效任务中的净收藏 Hook 数为分子、有效任务 Hook 数为分母；测试内选择率以有效任务中的最终首选 Hook 数为分子、有效任务 Hook 数为分母。无效任务必须同时报告排除的任务数和 Hook 数。

## 待数据填充的结论结构

### 样本、时间和 Prompt 版本

待产品负责人提供真实 CSV 后，由脚本输出总参与者、总任务、总 Hook、有效样本、排除样本、时间范围和 Prompt 版本。报告只引用脚本计算值。

### 收藏率与测试内选择率

待填充整数分子、分母和一位小数百分比。当前投递版写作“测试内采用率约 9%”，在数据核对前应删除；即使未来得到验证，对外名称也只能使用指标字典规定的“测试内最终选择率”。若计算值与简历不同，以计算值为准，并由产品负责人同步简历。

### Bad Case 与护栏

待按有效 Hook 汇总 `BAD_CASE_TYPES` 和护栏触发次数。多标签百分比的合计可能超过 100%，必须在结果旁说明。

### 证据与 Prompt 升级决定

受控测试数据通过校验且产品负责人核对分子、分母和来源后，只升级受控测试相关证据状态。该数据不能单独形成 Prompt 升级结论；Prompt 是否升级仍要求完整 60 案例正式盲评。离线提升值继续保持 `not_verified`。
