# 离线评测证据状态

## 已验证内容

- `eval/topics.json` 包含 20 个固定主题。
- `lib/evaluation/seeds.ts` 为每个主题生成小红书、抖音、B 站 3 个案例，共 60 个固定案例。
- 系统实现 baseline/candidate、正式结果选择、双人盲评、分歧裁决、Bad Case 和来源隔离。

## 尚未验证内容

本次审计时，本地 evaluation store 包含 60 个案例但没有正式评测 run；`eval/results` 仅有一次 10 条 Hook 的样例输出，人工计分数为 0。因此不能确认 candidate 可用率或平台适配率的任何提升值，也不能基于该批次做 Prompt 升级决定。

## 正式运行要求

1. 固定 baseline/candidate 内容哈希与模型参数。
2. 生成完整 60 案例的两组结果并记录格式失败。
3. 两名评测者独立盲评，分歧由第三人裁决。
4. 导出原始评分、聚合报告和 Bad Case。
5. 使用统一脚本计算分子、分母、百分点变化和平台分层结果。
6. 只有完整批次满足主指标和护栏时才提出升级建议；Mock 或不完整批次不得升级。
