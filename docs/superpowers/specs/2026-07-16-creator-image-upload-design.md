# 创作台图片上传与豆包识别设计

## 背景

图片上传功能已经在隔离工作区 `.worktrees/doubao-image-upload` 中实现，但尚未提交，也没有同步到当前主工作区。该隔离工作区基于较早的代码版本，直接合并会撤销已上线的管理员数据看板、PostgreSQL 持久化、分析数据隐私过滤和创作台按钮选中态修复。

本次工作采用“选择性移植”方式：只提取图片上传、豆包识别、图片上下文生成和对应测试，再适配当前创作台，不合并旧工作区的导航、后台或分析代码。

## 目标

- 用户可在创作台上传一张内容截图。
- 豆包视觉模型自动识别主题并推荐平台、内容类型和情绪风格。
- 识别结果自动填入现有表单，所有字段仍可手动修改。
- 图片描述作为补充上下文进入现有 DeepSeek Hook 生成流程。
- 原图只在单次请求期间处理，不写入磁盘、数据库、localStorage、历史记录或分析事件。
- 未上传图片、清除图片或图片识别失败时，原有纯文字创作流程保持可用。

## 不在范围内

- 不建立图片素材库。
- 不保存或复用原图。
- 不支持一次上传多张图片。
- 不改变现有 DeepSeek 模型、管理员后台、评测流程或数据看板权限。
- 不公开恢复“离线评测”或“数据看板”导航。

## 方案选择

### 采用：选择性移植并适配当前代码

从隔离工作区提取图片相关实现，逐文件合入当前分支，并根据当前 API、分析隐私策略和组件样式进行调整。

优点：保留已有功能和修复；图片功能已有较完整测试；冲突范围可控。缺点：需要人工核对两个版本间的 `app/page.tsx`、`InputPanel.tsx` 和测试差异。

### 不采用：直接合并旧分支

旧分支停留在后台改造前的提交。直接合并会产生大量反向修改，并可能重新暴露后台入口或恢复匿名数据接口。

### 不采用：完全重新实现

可以避免搬运旧代码，但会重复已经完成的上传校验、超时处理、请求竞态处理和测试工作，风险与成本更高。

## 用户体验

### 初始状态

在“主题”输入框上方显示“内容截图（可选）”上传区。支持点击选择和拖拽上传，接受 JPEG、PNG、WebP，最大 5MB。页面明确说明图片会发送至豆包识别、文字描述用于 DeepSeek 生成、原图不会保存。

### 识别状态

选择图片后立即显示本地缩略图，并显示“正在识别”。识别期间禁用生成按钮，防止图片描述尚未返回时提前生成。用户上传新图或清除图片时，中止前一个识别请求，过期请求不得覆盖最新表单状态。

### 识别成功

自动填入：

- 主题；
- 发布平台；
- 内容类型；
- 情绪风格。

推荐项显示“荐”角标。自动填入后所有字段维持普通可编辑状态，用户的后续手动选择拥有最终决定权。

### 清除与失败

清除图片会撤销缩略图、识别结果和图片描述，但不会回滚用户当前表单内容。识别失败时显示可理解的错误信息，保留缩略图和原有手动输入能力；用户可以更换图片、清除图片或直接继续纯文字创作。

## 架构与数据流

1. 浏览器选择图片并创建仅用于预览的 Object URL。
2. 浏览器以 `multipart/form-data` 将原始 `File` 发送到 `/api/analyze-image`。
3. 服务端验证文件大小、MIME 类型和文件签名。
4. 服务端在内存中转换为 data URL，调用豆包 Ark Chat Completions API。
5. 豆包按严格 JSON Schema 返回主题、图片描述和三个推荐值。
6. 服务端校验响应字段、枚举和长度后返回 `ImageAnalysisResult`。
7. 浏览器自动填入表单；生成时只把经校验的 `imageDescription` 传给 `/api/generate`。
8. `/api/generate` 对图片描述再次执行类型、长度和敏感信息检查，再将它注入 DeepSeek 用户提示词。
9. 生成响应、历史记录和分析事件不包含图片、data URL 或图片描述。

## 模块边界

### `lib/imageAnalysis.ts`

负责纯服务端图片校验、豆包请求、严格响应解析和错误映射。路由只负责注入环境变量，便于在不调用真实豆包的情况下测试核心行为。

### `app/api/analyze-image/route.ts`

Node.js Route Handler。读取 `ARK_API_KEY` 与 `ARK_MODEL_ID`，调用图片分析处理器，不记录请求体、data URL、密钥或上游完整响应。

### `components/InputPanel.tsx`

只负责上传交互、预览和推荐展示。文件读取、请求生命周期与识别结果状态由页面管理，避免组件同时承担网络与表单业务逻辑。

### `app/page.tsx`

管理当前图片、预览 URL、识别状态、取消控制器和请求序号。识别成功后填充现有表单状态；生成时传递图片描述。必须保留当前 `generation_start` 分析事件不包含原始主题。

### `lib/promptTemplates.ts` 与 `/api/generate`

`GenerateRequest` 增加可选 `imageDescription`。服务端将其标准化后作为“图片参考”加入提示词；纯文字请求生成的提示词保持不变。

## API 契约

### `POST /api/analyze-image`

请求：`multipart/form-data`，字段 `image` 为单个文件。

成功响应：

```json
{
  "topic": "三步写好产品周报",
  "imageDescription": "一张产品周报教程截图，包含目标、进展和风险三个模块。",
  "suggestedPlatform": "xiaohongshu",
  "suggestedContentType": "tutorial",
  "suggestedEmotionTone": "authoritative"
}
```

错误状态：

- `400`：请求格式、文件类型、文件签名或分析结果不合法；
- `413`：图片超过 5MB；
- `429`：豆包限流；
- `501`：生产环境未配置豆包凭据或模型接入点；
- `502`：豆包鉴权失败、服务异常或响应不符合契约；
- `504`：15 秒内未完成识别。

所有错误响应保持 `{ "error": string, "message": string }` 结构。

## 安全与隐私

- 只接受 JPEG、PNG、WebP，并同时验证 MIME 与魔数，拒绝伪装文件。
- 原始文件最大 5MB；图片描述最大 500 字符；生成路由采用更严格的业务长度限制。
- 图片中的文字一律视为待分析素材，不作为系统指令执行，降低图片提示词注入风险。
- 豆包输出使用严格 JSON Schema，并在服务端再次执行字段、枚举与长度白名单校验。
- 不将图片、data URL、图片描述或原始主题写入数据看板事件。
- 不把 Ark 密钥写入源代码、日志、文档或客户端包。
- 用户已经在聊天中提供过 Ark 密钥；部署完成后应在豆包控制台轮换，并只在 Vercel Production 环境保存新密钥。

## 环境配置

生产环境需要两个加密变量：

- `ARK_API_KEY`：豆包 Ark API 密钥；
- `ARK_MODEL_ID`：支持视觉输入的 Ark 推理接入点 ID。

缺少任一变量时，图片分析接口返回 `501`，但纯文字创作与已上线功能继续工作。部署前必须取得实际 `ARK_MODEL_ID`；代码和设计文档不保存其值。

## 错误处理与并发

- 客户端使用 `AbortController` 取消被替换或清除图片的请求。
- 每个识别请求分配递增序号，只有最新请求可更新界面。
- Object URL 在替换图片、清除图片和页面卸载时释放。
- 上传期间或生成期间禁用会产生冲突的操作。
- 上游错误向用户显示稳定中文信息，不转发豆包响应正文或敏感配置。

## 测试策略

### 服务端行为

- 接受真实签名的 JPEG、PNG、WebP；
- 拒绝空文件、超限文件、不支持类型和 MIME/签名不一致文件；
- 验证 multipart 请求和不配置环境变量的降级；
- 验证豆包请求使用视觉模型接入点、严格 JSON Schema、超时信号和无持久化字段；
- 验证豆包鉴权、限流、服务异常、无效 JSON、缺字段、额外字段、非法枚举与超长内容。

### 提示词与生成

- 纯文字请求提示词保持现状；
- 合法图片描述只作为图片参考注入一次；
- 超长、错误类型或包含敏感个人信息的图片描述在服务端被拒绝；
- 生成响应和历史数据不携带图片描述。

### 创作台契约

- 上传入口、文件类型、5MB 提示、隐私说明、识别状态与清除按钮存在；
- 自动推荐角标存在；
- 识别结果可继续手动修改；
- 取消过期请求和释放预览 URL；
- 保留按钮 `aria-pressed` 选中态、管理员导航隐藏和分析事件不含原始主题。

### 回归验证

运行完整测试、ESLint、Next.js 生产构建，并在生产环境验证：

- 纯文字生成仍返回成功；
- 合法截图能够识别并自动填入；
- 用户修改自动填入值后，生成使用修改后的值；
- 清除图片后可正常纯文字生成；
- 管理员数据看板权限、PostgreSQL 持久化和公开导航策略保持不变。

## 完成标准

- 图片上传功能出现在当前创作台，而不是仅存在于隔离工作区。
- 图片识别、自动填入、手动修改、生成和清除形成完整闭环。
- 图片和图片描述不进入持久化或分析事件。
- 当前后台、数据库、评测、导航与按钮选中态没有回归。
- 自动化验证和生产 smoke test 通过。
