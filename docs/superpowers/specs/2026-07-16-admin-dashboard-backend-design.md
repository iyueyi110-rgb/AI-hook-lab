# 管理员数据看板后端化设计

日期：2026-07-16

## 目标

将当前公开的 `/dashboard` 数据看板迁入受保护的管理后台，并使用云端 PostgreSQL 持久保存真实用户事件、管理员账号、会话和评测数据。

本次改动同时解决两个问题：

1. 看板页面和汇总接口当前可被任何访客读取。
2. Vercel 生产环境未配置 `DATABASE_URL` 时会降级到临时文件，数据无法稳定保存。

## 已确认的产品决策

- 后台地址使用 `/admin/dashboard`。
- 公开导航移除“数据看板”入口。
- 旧 `/dashboard` 自动跳转到 `/admin/dashboard`。
- 复用现有评测系统的账号、角色和会话机制。
- 只有 `admin` 角色可以读取看板；评测员和裁决员不能读取。
- 使用 Vercel Marketplace 的托管 Neon PostgreSQL，采用免费方案起步。
- 线上数据库从零开始，不迁移本机的一条测试事件。
- 首次打开登录页时创建首个管理员账号。
- 不保存用户输入的完整主题文本。

## 总体架构

### 公开产品

公开首页继续提供 Hook 生成、复制、收藏、采用和满意度评分。用户行为通过 `POST /api/dashboard/events` 发送到服务端。该接口仅允许写入受支持的事件，不提供数据读取能力。

事件写入失败不得阻断 Hook 生成或页面操作。浏览器端已有的本地统计仍可继续作为当前设备的即时反馈，但生产看板以 PostgreSQL 数据为准。

### 管理后台

`/admin/dashboard` 是服务端鉴权页面。访问时读取现有评测会话：

- 未登录：跳转到 `/evaluation/login?next=/admin/dashboard`。
- 已登录但角色不是 `admin`：显示 403 无权限状态。
- 管理员：读取并呈现看板汇总。

登录页支持经过白名单校验的站内 `next` 参数。登录或首次创建管理员成功后返回目标后台地址；外部 URL、协议相对 URL 和非允许路径不得成为跳转目标。

`/dashboard` 不再直接渲染数据，只执行到 `/admin/dashboard` 的服务端重定向。

### 看板 API

`GET /api/dashboard/summary` 复用相同的服务端会话校验：

- 无有效会话返回 401。
- 有效会话但不是管理员返回 403。
- 管理员请求返回按数据来源筛选的汇总结果。

前端刷新看板时继续调用该接口。页面首次加载也必须经过管理员校验，不能只依赖浏览器端隐藏入口。

### 数据库

Vercel Marketplace 的 Neon 集成向项目注入 PostgreSQL 连接配置，项目继续统一读取 `DATABASE_URL`。数据库承载：

- `dashboard_event` 及相关索引。
- 现有评测系统的状态表、账号投影、会话、案例、运行和审计表。

现有建表逻辑保持幂等：首次访问或显式初始化时创建表，重复运行不会删除或覆盖有效数据。

生产环境缺少 `DATABASE_URL` 时不得写入 Vercel 临时文件。事件接口应返回明确的服务不可用响应，管理员页面应显示数据库未配置错误。本地开发环境仍允许使用 JSON 文件降级，便于离线开发。

## 数据流

1. 用户在公开页面执行生成、复制、收藏、采用或评分操作。
2. 浏览器构造事件并调用写入接口。
3. 服务端验证事件类型、时间戳、载荷结构和最大大小。
4. 普通公开请求统一记为 `real_user`；只有携带正确 `EVAL_INGEST_TOKEN` 的内部请求可以写入 `evaluation_set` 或 `simulation`。
5. 服务端将规范化事件写入 Neon PostgreSQL。
6. 管理员登录后访问看板；服务端验证会话和管理员角色。
7. 看板按来源读取汇总，并可通过受保护接口刷新。

数据库保留全部事件。首版看板继续以最近 5,000 条事件作为统计窗口，避免一次性把无限增长的数据加载到应用内存；超出窗口的旧记录不会被删除。后续如果数据量增长，可将聚合下推为 SQL 查询并增加日期筛选，此项不在本次范围内。

## 数据最小化

当前 `generation_start` 事件包含完整主题文本，但现有看板并不使用该字段。改动后不再把完整主题写入分析事件。

允许保存的事件字段包括：

- 平台和内容类型。
- 模型、Prompt 版本和变体。
- Hook 数量、平均评分、点击欲望评分和 bad case 标签。
- 生成耗时和错误类别。
- Hook 随机标识、收藏、复制、采用和满意度评分。

接口继续限制 payload 大小。管理员看板不展示密码、会话令牌、数据库连接串或 API 密钥。

## 页面与导航

- 公开 `AppHeader` 移除数据看板链接。
- 管理员在评测工作区和数据看板之间可以通过内部后台导航切换。
- 看板现有指标、来源筛选、刷新和最近事件展示保持不变，本次不重做视觉设计。
- 登录页文案同时覆盖“内部账号登录”和“创建首个管理员”，登录目标由安全的站内 `next` 参数决定。

## 错误处理

- 数据库未配置：生产环境明确报错，不使用临时文件制造“已持久化”的假象。
- 数据库暂时不可用：公开产品的核心生成流程继续工作；事件上报失败不阻断用户操作。
- 看板读取失败：保留现有已加载数据并显示错误，不把读取失败解释成零数据。
- 无效事件、超大 payload 或不支持的数据来源：返回 4xx，不写入数据库。
- 会话过期：受保护页面跳转登录，API 返回 401。
- 权限不足：页面和 API 均返回 403 语义。

## 测试与验收

### 自动化测试

- 未登录访问管理员看板会跳转登录页并携带安全的站内返回地址。
- 管理员可访问页面和汇总接口。
- 评测员和裁决员不能访问页面或汇总接口。
- 汇总 API 对无会话返回 401，对非管理员返回 403。
- 公开事件接口无需管理员会话即可写入合法真实用户事件。
- 普通请求不能伪造 `evaluation_set` 或 `simulation` 来源。
- 完整主题不会进入服务端分析事件 payload。
- 站外或畸形 `next` 参数不会造成开放重定向。
- 生产环境缺少 `DATABASE_URL` 时不会降级写入临时文件。
- 建表和初始化可重复执行。

### 上线验收

1. 在 Vercel Marketplace 创建并绑定 Neon 数据库。
2. 确认生产环境存在加密的 `DATABASE_URL`，不输出其值。
3. 部署新版并确认状态 Ready。
4. 访问 `/admin/dashboard`，验证未登录跳转。
5. 创建首个管理员并确认登录后返回看板。
6. 在公开首页完成一次真实 Hook 生成和至少一次操作事件。
7. 刷新管理员看板，确认真实用户数据增加。
8. 直接请求汇总 API，确认未登录和非管理员请求被拒绝。
9. 验证 `hookovo.icu` 首页、Hook 生成和 HTTPS 仍正常。

## 不在本次范围内

- 迁移本机测试事件或旧的临时 Vercel 文件。
- 新增多租户、组织或细粒度权限系统。
- 重做看板视觉样式。
- 增加日期范围、导出、定时报告或实时推送。
- 将本机 PostgreSQL 暴露到公网。
